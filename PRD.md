# 레퍼런스랩 (ReferenceLab) — PRD

## 1. 서비스 개요
레퍼런스랩은 콘텐츠 마케터와 PD를 위한 레퍼런스 분석·수집 SaaS 도구입니다.
인스타그램 링크를 입력하면 후킹포인트를 자동 분석하고, 핀터레스트 보드의 이미지를 피그마로 자동 수집해 광고 소재 제작 효율을 높입니다.

## 2. 폴더 구조
이 레포는 5개 팀(파트)에 따른 5개 영역으로 나뉜다:
- planning/ — 기획 (무엇을 만들 것인가)
- design/ — 디자인 (어떻게 보여줄 것인가)
- cx/ — CX (누가 어떻게 사용할 것인가)
- marketing/ — 마케팅 (어떻게 알릴 것인가)
- strategy/ — 전략 (어떻게 지속 가능하게 만들 것인가)

각 영역의 상세는 해당 폴더에서 작성된다.

## 3. 기획 (Planning)
- 인스타그램 링크 입력 → 이미지/영상 후킹포인트 분석
- 핀터레스트 보드 이미지 → 피그마 자동 수집
- MVP 범위: 두 가지 핵심 기능 중심으로 먼저 출시

상세는 `planning/` 폴더 참조. [작성 예정]

## 4. 디자인 (Design)
분석 결과를 직관적으로 시각화하는 대시보드 중심의 UI
링크 입력 → 분석 결과 → 저장/내보내기의 단순한 화면 흐름

상세는 `design/` 폴더 참조. [작성 예정]

## 5. CX (Customer Experience)
타겟 사용자: 광고 소재를 주로 제작하는 콘텐츠 마케터 및 PD
사용자 여정: 링크 입력 → 분석 결과 확인 → 레퍼런스 저장 → 피그마 활용

상세는 `cx/` 폴더 참조. [작성 예정]

## 6. 마케팅 (Marketing)
업무용 내부 도구로 시작, 팀 내 공유를 통한 사용자 확대
주요 채널: 팀 내 도입 → 입소문

상세는 `marketing/` 폴더 참조. [작성 예정]

## 7. 전략 (Strategy)
수익 모델 없음 (업무용 내부 도구)
안정적인 업무 효율 향상을 목표로 지속적인 기능 개선

상세는 `strategy/` 폴더 참조. [작성 예정]

## 8. 역할 배정 (가상)
이 프로젝트를 팀으로 한다면:

| 팀 | 담당 영역 | 작업 폴더 |
|----|---------|----------|
| PM (본인) | 전체 기획·의사결정 | 루트 |
| 기획 팀 | 기능 정의, MVP 범위 | planning/ |
| 디자인 팀 | UI/UX, 비주얼 | design/ |
| CX 팀 | 페르소나, 여정맵, FAQ | cx/ |
| 마케팅 팀 | 런칭, 채널 | marketing/ |
| 전략 팀 | 수익 모델, 성장 | strategy/ |
